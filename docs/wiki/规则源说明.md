## 规则源分类逻辑

### 为什么分手机端和电脑端？

移动端和桌面端的广告形式不同：

- **手机端** — 移动 App 内广告、移动网页广告、省流量模式下的广告
- **电脑端** — 桌面浏览器网页广告、弹窗、视频广告、社交媒体嵌入

分开维护可以让规则更精准，避免手机端加载不必要的电脑端规则（反之亦然）。

### `bo`（两端共用）的工作原理

`category: bo` 的源会：

1. 展开为 mobile 和 pc 两条记录
2. URL 相同时只下载一次（去重）
3. 下载结果同时分发给 mobile 和 pc 的合并流程

这样既保证了两端都能用到这些规则，又避免了重复下载。

## 如何新增规则源

### 第一步：选择一个可靠的规则源

规则源需要满足：

- 返回纯文本格式（ABP/uBlock/AdGuard 格式）
- 可直链访问（不需要登录、不强制跳转）
- 持续维护（长期不更新的规则源效果会下降）

### 第二步：编辑 `config/sources.yaml`

在 `sources:` 下新增一项，复制模板改 3 处即可：

```yaml
  # 你的规则说明（可选但推荐）
  - name: 你的规则名
    category: mo          # mo=手机端 / pc=电脑端 / bo=两端共用
    url: https://你的规则地址
    id: x1                # 不能与已有 id 重复
```

### 第三步：验证

```bash
# 抓取测试
python scripts/fetch_rules.py

# 检查 fetch_meta.json（临时目录）中的结果
```

## 如何禁用规则源

将整个源项用 `#` 注释掉：

```yaml
  # - name: 不需要的源
  #   category: mo
  #   url: https://example.com/filter.txt
  #   id: m2
```

或者删除该项（注释更安全，方便日后恢复）。

## 规则格式说明

FilterFusion 支持以下格式的规则：

| 格式 | 示例 | 说明 |
|------|------|------|
| ABP 标准 | `||ads.example.com^` | 最常用，uBlock Origin、AdGuard 均支持 |
| 元素隐藏 | `example.com##.ad-banner` | 隐藏页面元素 |
| 例外规则 | `@@||example.com^` | 排除特定域名 |
| DNS 规则 | `||ads.example.com^` | DNS 过滤格式，仅用于 DNS 规则源 |
