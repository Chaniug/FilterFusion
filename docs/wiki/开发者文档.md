# 开发者文档

本文档介绍 FilterFusion 的架构设计、脚本说明和贡献指南。

## 项目架构

```
FilterFusion/
├── config/                      # 配置文件
│   ├── sources.yaml             # AdBlock 规则源配置
│   ├── dns_sources.yaml         # DNS 规则源配置
│   ├── default.header          # AdBlock 规则头部模板
│   ├── dns.header             # DNS 规则头部模板
│   ├── summary.json           # 统计摘要（控制台输出，不生成文件）
│   └── dns_summary.json      # DNS 统计摘要（控制台输出，不生成文件）
├── scripts/                     # 核心脚本
│   ├── base_fetcher.py        # 共享抓取基类
│   ├── fetch_rules.py         # AdBlock 规则抓取
│   ├── fetch_dns_rules.py    # DNS 规则抓取
│   ├── fetch_all.py          # 统一抓取入口
│   ├── merge_rules.py        # AdBlock 规则合并去重
│   ├── merge_dns_rules.py    # DNS 规则合并去重
│   ├── merge_all.py          # 统一合并入口
│   └── __init__.py          # Python 包标识
├── dist/                       # 生成的规则文件（Git 追踪）
│   ├── adblock-mo.txt        # 手机端规则
│   ├── adblock-pc.txt        # 电脑端规则
│   └── dns-blocklist.txt     # DNS 过滤规则
├── docs/                       # 项目文档
│   ├── PROJECT_DOCS.md       # 完整项目文档
│   ├── PROJECT_LOG.md        # 开发日志
│   └── wiki/                # GitHub Wiki 页面
├── assets/                     # 静态资源
├── .github/workflows/          # GitHub Actions 工作流
│   ├── daily-update.yml      # 每日自动更新（含 Pages 部署）
│   └── weekly-release.yml    # 每周自动发布
├── requirements.txt            # Python 依赖
├── pyproject.toml             # 现代 Python 项目配置
└── README.md                  # 项目说明
```

## 核心脚本详解

### `base_fetcher.py` — 共享抓取基类

**职责**：定义 `BaseFetcher` 基类，封装共享的抓取逻辑。

**关键设计**：

- `FetchStatus(StrEnum)` — 抓取状态枚举（`"success"` / `"failed"` / `"disabled"`）
- `fetch_single_rule()` — 单个规则源抓取（含重试、超时、错误处理）
- `fetch_all_rules()` — 批量抓取（按 URL 去重，同 URL 只下载一次）
- `_build_success()` / `_build_failure()` / `_build_disabled()` — 构建元数据

**子类只需实现**：

```python
def load_sources(self) -> list[SourceInfo]:
    # 解析配置文件，返回源列表
    pass
```

### `fetch_rules.py` — AdBlock 规则抓取

**职责**：从 `config/sources.yaml` 读取 AdBlock 规则源，并发抓取。

**关键流程**：

1. `load_sources()` — 解析 YAML，提取 `sources` + `custom_rules`
2. ID 冲突检测 — 重复 ID 报错并跳过
3. `category: bo` 展开 — 展开为 mobile + pc 两条记录
4. 并发抓取 — `httpx.AsyncClient` + `asyncio.gather()`
5. 写入元数据 — `fetch_meta.json`（临时目录）

**网络层优化**：

- `httpx[http2]` — HTTP/2 多路复用
- 同在 `raw.githubusercontent.com` 的源共享单条 TLS 连接
- 超时 35-65s（适配大型规则文件）

### `fetch_dns_rules.py` — DNS 规则抓取

**职责**：从 `config/dns_sources.yaml` 读取 DNS 规则源，抓取并写入本地。

**与 AdBlock 抓取的区别**：

- 无 `category`、`id` 字段
- 不支持组合规则
- 合并逻辑更简单（仅去重）

### `fetch_all.py` — 统一抓取入口

**职责**：在单进程单事件循环中同时执行 AdBlock + DNS 抓取。

**优势**：

- 消除 Shell 后台任务开销
- 共享 `asyncio` 事件循环
- 代码更简洁

### `merge_rules.py` — AdBlock 规则合并去重

**职责**：将抓取的多个规则源按类别合并去重，输出标准 ABP 格式。

**关键设计**：

- `RuleType(StrEnum)` — 规则类型枚举
- `_SPECIAL_RE` / `_REGEX_CHAR_RE` / `_HTML_FILTER_RE` — 类级预编译正则
- `format_map()` + `_SafeDict` — 头部模板单次替换（替代 11 次链式 `.replace()`）
- 按类型分组 — 注释、异常规则、元素隐藏、普通规则分别输出

**CLI 参数**：

```bash
python scripts/merge_rules.py --category mobile|pc
```

### `merge_dns_rules.py` — DNS 规则合并去重

**职责**：将抓取的 DNS 规则源去重合并，输出 DNS 过滤格式。

**与 AdBlock 合并的区别**：

- 无复杂分类逻辑
- 仅例外规则 + 普通规则去重
- 输出格式为 AdGuard DNS 过滤格式

### `merge_all.py` — 统一合并入口

**职责**：单进程执行 DNS 合并 + AdBlock custom_rules 组合规则。

**优势**：

- 减少 Python 解释器启动开销（3 步 → 1 步）
- 对齐已有的 `fetch_all.py` 单进程模式

## 数据流

```
config/sources.yaml  (规则源配置)
        ↓
  fetch_rules.py  (并发抓取)
        ↓
  fetch_meta.json  (临时目录，运行时元数据)
        ↓
  merge_rules.py  (合并去重)
        ↓
  dist/adblock-mo.txt  (手机端规则)
  dist/adblock-pc.txt  (电脑端规则)

config/dns_sources.yaml  (DNS 规则源配置)
        ↓
  fetch_dns_rules.py  (抓取)
        ↓
  dns_fetch_meta.json  (临时目录)
        ↓
  merge_dns_rules.py  (合并去重)
        ↓
  dist/dns-blocklist.txt  (DNS 过滤规则)
```

## 技术决策

### 为什么用 YAML 而不是 JSON？

- **可读性** — YAML 支持注释，JSON 不支持
- **编辑友好** — GitHub 网页编辑有 YAML 语法高亮，手机端编辑友好
- **扩展性** — 新增字段只需加 key，不影响现有配置

### 为什么用 `httpx` 而不是 `requests`？

- **异步支持** — `httpx.AsyncClient` 原生支持 `async/await`
- **HTTP/2** — `httpx[http2]` 支持 HTTP/2 多路复用
- **性能** — 异步并发比多线程更高效

### 为什么元数据写入临时目录？

- **保持项目目录干净** — 不生成运行时缓存文件
- **安全性** — 临时文件运行后自动销毁
- **Git 友好** — 减少不必要的变更

## 贡献指南

### 代码风格

- **类型注解** — 所有函数必须有完整类型注解
- **StrEnum** — 字符串枚举使用 `StrEnum`
- **`__slots__`** — 数据类使用 `__slots__` 减少内存
- **`encoding='utf-8'`** — 所有 `open()` 必须指定编码

### 提交规范

```
<type>: <description>

例：
feat: 新增自定义组合规则功能
fix: 修复 weekly-release 规则数重复统计 bug
docs: 更新 README 订阅地址
```

### 测试

本地运行完整流程：

```bash
python -m scripts.fetch_all
python -m scripts.merge_all
```

检查 `dist/` 下的规则文件是否正确生成。
