# 配置说明

FilterFusion 的所有配置都放在 `config/` 目录下，采用 YAML 格式（对大小写敏感）。

## 配置文件一览

| 文件 | 用途 |
|------|------|
| `config/sources.yaml` | AdBlock 规则源配置（含自定义组合规则） |
| `config/dns_sources.yaml` | DNS 规则源配置 |
| `config/default.header` | AdBlock 规则文件头部模板 |
| `config/dns.header` | DNS 规则文件头部模板 |

## `sources.yaml` 详解

这是最核心的配置文件，控制"从哪里抓、怎么组合、输出什么"。

### 文件结构

```yaml
# ============================================================
# FilterFusion AdBlock 规则源配置
# ------------------------------------------------------------
# 格式: YAML（对大小写敏感，所有值建议用小写）
#
# 【字段说明】（每个源必填 4 项，顺序可任意）
#   name     : 规则源的名字（显示用，可中文）
#   category : 分类，取以下三个缩写之一：
#                mo = 手机端 (Mobile)
#                pc = 电脑端 (PC)
#                bo = 两端共用 (Both)，同 URL 只下载一次，自动分发给 mo 和 pc
#              （也兼容旧写法 mobile / pc / both）
#   url      : 规则文件的直链地址（必须以 http 开头）
#   id       : 这个源的唯一短编号（如 m1、b2、p1）
#              ★ id 是干嘛用的？
#                下面 custom_rules 段里要"按 id 引用源"来组合产出文件，
#                比如 sources: [m1, m2, b1] 就是把 m1、m2、b1 三个源合并成一个文件。
#                所以每个源都得有个 id，别人才能引用它。id 不能重复。
#
# 【启用 / 禁用】
#   写在 sources 列表里就是启用；整项前面加 # 注释掉就是禁用。
#
# 【给源写说明？】
#   在某个源上方加一行 # 注释即可（脚本会忽略），方便日后记住这个源是干嘛的。
#   例: # AdGuard 官方移动端过滤规则
#       - name: AdGuard Mobile
#
# 【新增一个源？复制下面模板，改 3 处即可】
#   - name: 你的规则名
#     category: mo          # mo=手机端 / pc=电脑端 / bo=两端共用
#     url: https://你的规则地址
#     id: x1                # 随便起个不重复的编号就行
# ============================================================

sources:
  # ===== 手机端 (Mobile) =====
  # AdGuard 官方移动端过滤规则（默认手机端拦截）
  - name: AdGuard Mobile
    category: mo
    url: https://raw.githubusercontent.com/...
    id: m1

  # ... 更多源 ...

# ============================================================
# 自定义组合规则（custom_rules）
# ------------------------------------------------------------
# 从上面已抓取的源里，按 id 挑选几个，合并去重后输出标准 ABP 格式到 dist/。
#
# 【字段说明】
#   output      : 输出文件名（保存到 dist/ 下）
#   sources     : 引用上方 sources 里定义的 id 列表
#   description : 可选，规则文件的描述文本（不填则自动生成）
#
# 【新增一条组合规则？复制下面模板，改 2 处即可】
#   - output: 我的规则.txt
#     sources: [m1, b1]      # 想合并哪些源，就填哪些 id
# ============================================================
custom_rules:
  - output: adblock-mo.txt
    description: FilterFusion - Ad-blocking rules (Mobile)
    sources: [m1, m2, m3, b1, b2]
```

### category 缩写说明

| 缩写 | 全称 | 含义 |
|------|------|------|
| `mo` | mobile | 手机端规则 |
| `pc` | pc | 电脑端规则 |
| `bo` | both | 两端共用（同 URL 只下载一次） |

> ⚠️ 同时兼容旧全称 `mobile` / `pc` / `both`，向后兼容无破坏。

### id 字段详解

`id` 是每个规则源的唯一短编号，有两个作用：

1. **被 `custom_rules` 引用** — 组合规则通过 id 来挑选要合并的源
2. **去重标识** — `category: bo` 的源会展开为两条记录，通过 id 共享下载结果

id 命名建议：
- 手机端：`m1`、`m2`、`m3`...
- 两端共用：`b1`、`b2`...
- 电脑端：`p1`、`p2`...

### custom_rules 组合规则

`custom_rules` 段定义所有 AdBlock 产出文件，每个组合规则含三个字段：

```yaml
custom_rules:
  - output: adblock-mo.txt          # 输出文件名
    description: 描述文本             # 可选，写入规则文件头部
    sources: [m1, m2, b1]          # 要合并的源 id 列表
```

**示例：新增一个自定义规则文件**

```yaml
  - output: exten.txt
    sources: [m1, m2, b1]
```

取消注释后，运行合并脚本就会生成 `dist/exten.txt`。

## `dns_sources.yaml` 详解

DNS 规则源配置更简单，只需 `name` 和 `url` 两个字段：

```yaml
sources:
  - name: AdGuard DNS
    url: https://raw.githubusercontent.com/...
```

DNS 源不支持组合规则，每个源独立输出。

## 头部模板

### `default.header`（AdBlock）

```html
[Adblock Plus 2.0]
! Title: {TITLE}
! Description: {DESCRIPTION}
! Version: {VERSION}
! TimeUpdated: {TIME_UPDATED}
! Homepage: https://github.com/Chaniug/FilterFusion
! Checksum: {CHECKSUM}
```

`{TITLE}`、`{DESCRIPTION}` 等占位符在合并时自动替换。

### `dns.header`（DNS）

DNS 头部模板格式不同，适用于 DNS 过滤工具（如 AdGuard Home、Pi-hole）。

## YAML 注意事项

1. **大小写敏感** — `mo` ≠ `Mo` ≠ `MO`，建议全用小写
2. **缩进必须用空格** — 不能用 Tab
3. **冒号后加空格** — `name: 值` ✅，`name:值` ❌
4. **URL 必须加引号** — 如果含特殊字符（如 `#`），需用引号包裹
