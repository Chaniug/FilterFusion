## CI 修复：uv pip install --system 错误

`daily-update.yml` 中 `uv pip install --system` 报错 "No system Python installation found for Python 3.13"。原因是 `setup-uv@v6` 安装的 Python 由 uv 管理而非系统级。修复：
- 去掉 `--system` 标志
- `python -m scripts.xxx` 改为 `uv run python -m scripts.xxx`

## CI 修复：完整迭代过程（4 轮报错 → 最终修复）

GPU Actions CI 由于最新优化（Python 3.13 + uv）触发连环报错，经多轮迭代全部修复：

1. **`--system` 报错**: `uv pip install --system` → 无系统 Python 3.13 → 改为 `uv venv && uv pip install -r requirements.txt`
2. **`uv sync` 构建失败**: setuptools 发现多个顶层目录（rules/assets/config）→ 不构建项目，仅安装依赖
3. **`uv run` 构建失败**: `uv run` 仍尝试构建项目 → 加 `--no-project` 跳过构建
4. **pyproject.toml 修复**: `license = {text = "MIT"}` → `license = "MIT"`（消除 SetuptoolsDeprecationWarning）；新增 `[tool.setuptools.packages.find]` 配置 `include = ["scripts*"]`，仅将 scripts 识别为 Python 包
5. **Node 20 弃用警告**: `astral-sh/setup-uv@v6` → `@v8.2.0`（v8.2.0 使用 node24；v8 短标签不存在需用完整版本号）

最终 daily-update.yml 依赖安装流程：`uv venv` → `uv pip install -r requirements.txt` → `uv run --no-project python -m scripts.xxx`
